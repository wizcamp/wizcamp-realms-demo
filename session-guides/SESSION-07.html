üß© Understanding Component Composition

Before we start coding, let‚Äôs understand how complex components are built from smaller pieces ‚Äî the foundation of scalable React architecture.

Component composition is like building with LEGO blocks ‚Äî you create complex structures by snapping together smaller, focused pieces. Your QuizModal is actually composed of five smaller components working together:

QuizModal (the container)
‚îú‚îÄ‚îÄ ProgressHeader (shows question progress)
‚îú‚îÄ‚îÄ QuestionHeader (displays the question)
‚îú‚îÄ‚îÄ AnswerChoices (interactive answer buttons)
‚îú‚îÄ‚îÄ AnswerFeedback (shows results)
‚îî‚îÄ‚îÄ ContinueButton (navigation control)

üí° Why This Matters

Component composition is how professional React apps stay organized and maintainable. Instead of one massive component doing everything, you break functionality into focused pieces. Each component has a single responsibility, making your code easier to understand, test, and modify.

  4. Transform the answers array into buttons using React‚Äôs mapping pattern:

    function AnswerChoices({ answers }) {
  return (
    <div className="answers-grid">
      {answers.map((answer, index) => ( // Turn answers into buttons
        <button key={index} className="answer-button">
          {answer}
        </button>
      ))}
    </div>
  );
}

    Array mapping transforms each item in an array into something else. Here, each answer string becomes a button element. React needs a unique key prop for each mapped element to track changes efficiently.

    Test: Click zone ‚Üí You should see answer buttons appear in the modal

üí° Why This Matters

By starting with a simple structure and then adding the mapping logic, you followed a professional development pattern: build incrementally and test each step. Array mapping is everywhere in React ‚Äî any time you have a list of data that becomes a list of components, you use map(). The key prop helps React optimize updates by tracking which items changed, moved, or were added/removed, making your dynamic button lists performant and reliable.

    function AnswerChoices({ answers, onAnswerClick, chosenAnswer, correctAnswer }) {
  const getButtonStyle = (answerIndex) => {

    if (answerIndex === correctAnswer) return "answer-button correct";

üí° Why This Matters

Event handlers like onClick connect user actions to your app‚Äôs logic, making static components come alive with interactivity. Conditional styling then provides immediate visual feedback to users through the getButtonStyle function, which returns different CSS classes based on the current state. The disabled attribute prevents multiple clicks after an answer is selected, creating a polished user experience where buttons respond intelligently to user interactions.
